## MDX 完全指南 - 当 Markdown 遇上 JSX

> Markdown 给了我们书写的自由，JSX 赋予了我们表达的力量。MDX 将两者融为一体，让内容创作突破了纯文本的边界。

<!--more-->

## 一、MDX 与传统 Markdown 的对比分析

### 1.1 什么是 Markdown？

Markdown 是 John Gruber 于 2004 年创建的轻量级标记语言，设计初衷是让人们"用易读易写的纯文本格式编写文档，然后将其转换为有效的 HTML"。它以简洁的语法著称，几行符号就能表达标题、列表、链接、代码块等常见结构。

**Markdown 的核心特征：**

- 纯文本格式，人类可直接阅读
- 语法简洁，学习成本极低
- 专注于内容本身，而非排版细节
- 跨平台通用，几乎所有编辑器都支持

### 1.2 什么是 MDX？

MDX 是 Markdown 的超集，它允许你在 Markdown 文档中直接嵌入 JSX（JavaScript XML）语法。简单来说，**MDX = Markdown + JSX**。

这意味着你可以在一篇 Markdown 文章里直接使用 React 组件、Vue 组件，或任何框架的组件，实现传统 Markdown 无法完成的交互功能。

### 1.3 语法结构对比

#### 基础语法 - 两者完全一致

MDX 完全兼容 Markdown 的所有标准语法。以下内容在 Markdown 和 MDX 中写法完全相同：

```markdown
# 一级标题
## 二级标题

**加粗文本** 和 *斜体文本*

- 无序列表项 1
- 无序列表项 2

1. 有序列表项 1
2. 有序列表项 2

[链接文字](https://example.com)

![图片描述](./image.png)

> 引用文本

`行内代码` 和代码块：

```javascript
const greeting = "Hello, World!";
```
```

#### JSX 组件嵌入 - MDX 独有

MDX 的核心差异在于可以直接使用 JSX 组件：

```jsx
import { Chart } from './components/Chart';
import { Alert } from './components/Alert';

# 我的数据报告

这是一段普通的 Markdown 文本。

<Alert type="warning">
  注意：以下数据仅用于演示。
</Alert>

## 销售趋势

<Chart
  data={[12, 19, 3, 5, 2, 3]}
  labels={['一月', '二月', '三月', '四月', '五月', '六月']}
/>

数据来源于 2026 年第一季度报告。
```

#### JSX 表达式 - MDX 独有

在 MDX 中可以使用花括号 `{}` 嵌入 JavaScript 表达式：

```jsx
export const name = "MDX";
export const year = 2026;

# 欢迎学习 {name}

当前年份是 **{year}**。

{2 + 3 > 4 && '数学表达式可以直接计算'}
```

#### 自定义组件映射 - MDX 独有

MDX 支持通过 `components` 属性将 Markdown 原生元素映射为自定义组件：

```jsx
// 为所有标题添加锚点链接
// 为所有代码块添加复制按钮
const components = {
  h1: (props) => <h1 id={slugify(props.children)} {...props} />,
  pre: (props) => <CodeBlock copyable {...props} />,
};

<MDXContent components={components} />
```

### 1.4 功能特性对比

| 特性 | Markdown | MDX |
|------|----------|-----|
| 标准 Markdown 语法 | ✅ 完全支持 | ✅ 完全支持 |
| HTML 标签嵌入 | ⚠️ 部分支持 | ✅ 完全支持 |
| JSX 组件使用 | ❌ 不支持 | ✅ 原生支持 |
| JavaScript 表达式 | ❌ 不支持 | ✅ 原生支持 |
| 组件导入 | ❌ 不支持 | ✅ `import` 语法 |
| 变量与导出 | ❌ 不支持 | ✅ `export` 语法 |
| 交互式内容 | ❌ 不支持 | ✅ 可嵌入任意交互组件 |
| 数学公式 | ✅ 通过扩展 | ✅ 通过扩展或组件 |
| 代码高亮 | ✅ 通过扩展 | ✅ 通过扩展或组件 |
| 编译为 HTML | 直接转换 | 通过 MDX 编译器 |
| 构建复杂度 | 极低 | 中等 |
| 学习曲线 | 极低 | 低（需了解 JSX 基础） |

### 1.5 使用场景对比

**传统 Markdown 更适合：**

- 简单的文档编写（README、CHANGELOG）
- 纯文本内容为主的文章
- 不需要交互功能的静态页面
- 快速笔记和草稿
- 对构建工具要求极简的场景

**MDX 更适合：**

- 技术博客（嵌入代码演示、交互式示例）
- 组件文档（展示组件用法和效果）
- 产品文档（含交互式配置器、计算器）
- 教程和课程（含可交互的练习和测验）
- 任何需要"内容 + 交互"融合的场景

## 二、MDX 的应用价值与优势解析

### 2.1 组件化内容创作

MDX 最大的价值在于实现了**内容与组件的无缝融合**。

传统方式下，"文档内容"和"交互组件"是分离的：Markdown 负责文字，组件负责交互，两者需要在不同文件中维护。MDX 将它们统一到一个文件里：

```jsx
import { Button } from '../components/Button';
import { CodeSandbox } from '../components/CodeSandbox';

# 如何使用按钮组件

点击下方按钮查看效果：

<Button variant="primary" size="large">
  点击我
</Button>

你也可以在 CodeSandbox 中在线试用：

<CodeSandbox id="button-demo-abc123" />
```

这种模式的好处是：**内容创作者不需要在文档和代码之间来回切换**，一切都在同一个上下文中完成。

### 2.2 动态交互实现

MDX 让静态文档拥有了"生命力"：

```jsx
import { useState } from 'react';
import { Counter } from '../components/Counter';

# 交互式计数器

下面的计数器是完全动态的，试试点击：

<Counter initial={0} step={1} />

通过这种方式，读者可以在阅读文档的同时"动手体验"。
```

**典型交互场景包括：**

- **实时代码演示**：嵌入 CodeSandbox、StackBlitz 等在线编辑器
- **数据可视化**：嵌入图表、地图、3D 模型
- **交互式表单**：配置生成器、计算器、问卷
- **状态管理示例**：展示 React/Vue 的状态管理用法

### 2.3 前端框架集成便利性

MDX 生态对主流前端框架提供了优秀的支持：

**Astro**（本博客使用的框架）：

```javascript
// astro.config.mjs
import mdx from '@astrojs/mdx';

export default defineConfig({
  integrations: [mdx()],
});
```

Astro 原生支持 MDX，配置极其简单，且 MDX 页面与 `.astro` 页面享有完全相同的功能。

**Next.js**：

```javascript
// next.config.mjs
import createMDX from '@next/mdx';

const nextConfig = {
  pageExtensions: ['js', 'jsx', 'md', 'mdx'],
};

export default createMDX()(nextConfig);
```

**Gatsby**：通过 `gatsby-plugin-mdx` 插件集成。

**VitePress / VuePress**：通过插件或自定义配置支持。

### 2.4 内容复用性提升

MDX 的组件化特性天然支持内容复用：

**复用 UI 组件**：同一个 `Callout` 组件可以在所有文档中使用：

```jsx
import { Callout } from '../components/Callout';

<Callout type="info">
  这是一个信息提示框。
</Callout>

<Callout type="warning">
  这是一个警告提示框。
</Callout>
```

**复用数据**：通过 `export` 导出的数据可在其他模块中引用：

```jsx
export const metadata = {
  author: 'Kinyou',
  version: '2.0',
  lastUpdated: '2026-05-18',
};

# API 文档 v{metadata.version}
```

**复用布局**：通过 Layout 组件统一页面结构：

```jsx
import { DocLayout } from '../layouts/DocLayout';

export default function MDXLayout({ children }) {
  return <DocLayout sidebar>{children}</DocLayout>;
}

# 文档标题

正文内容自动应用 DocLayout 的样式和布局。
```

### 2.5 类型安全与开发体验

MDX 支持 TypeScript，可以在编译时发现错误：

```typescript
// mdx-components.tsx (Next.js 示例)
import type { MDXComponents } from 'mdx/types';

export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    h1: ({ children }) => <h1 className="text-3xl font-bold">{children}</h1>,
    ...components,
  };
}
```

配合编辑器的类型提示，可以获得与编写普通 TypeScript 代码一致的开发体验。

### 2.6 SEO 与可访问性

MDX 在编译时生成标准 HTML，输出结果与普通 Markdown 编译产物在 SEO 和可访问性上没有差异。搜索引擎可以正常抓取和索引 MDX 页面的内容。

## 三、MDX 实用指南与常用工具介绍

### 3.1 快速上手

#### 最简方式：在 Astro 中使用 MDX

如果你的项目已经使用 Astro（如本博客），只需三步：

**第一步**：确保安装了 MDX 集成

```bash
pnpm add @astrojs/mdx
```

**第二步**：在 `astro.config.mjs` 中启用

```javascript
import { defineConfig } from 'astro/config';
import mdx from '@astrojs/mdx';

export default defineConfig({
  integrations: [mdx()],
});
```

**第三步**：在 `src/content/posts/` 目录下创建 `.mdx` 文件

```bash
# 手动创建
touch src/content/posts/my-first-mdx.mdx

# 或使用项目脚本（需修改扩展名）
pnpm new-post -- my-first-mdx.mdx
```

然后像写 Markdown 一样写入内容，需要时加入 JSX 组件即可。

#### 独立使用：Node.js 项目

```bash
mkdir my-mdx-project && cd my-mdx-project
npm init -y
npm install @mdx-js/mdx @mdx-js/react react react-dom
```

创建一个简单的编译脚本：

```javascript
// compile.mjs
import { compile } from '@mdx-js/mdx';
import { readFileSync, writeFileSync } from 'fs';

const source = readFileSync('content.mdx', 'utf-8');
const result = await compile(source);
writeFileSync('output.mjs', String(result));

console.log('编译完成！');
```

### 3.2 核心语法规则

#### 导入组件

```jsx
// 具名导入
import { Button } from '../components/Button';

// 默认导入
import Callout from '../components/Callout';

// 导入非组件资源
import styles from './doc.module.css';
```

**注意**：导入语句必须放在文件顶部，在任何 Markdown 内容之前。

#### 使用组件

```jsx
// 自闭合标签
<Button />

// 带属性
<Button variant="primary" size="lg">点击</Button>

// 嵌套内容
<Callout type="info">
  这里可以包含 **Markdown** 语法。
</Callout>
```

#### JSX 表达式

```jsx
export const items = ['苹果', '香蕉', '橘子'];

水果列表：
{items.map((item, i) => (
  <span key={i}>{item} </span>
))}

计算结果：{Math.sqrt(144)}
```

#### 导出数据

```jsx
// 导出元数据（可被其他模块引用）
export const frontmatter = {
  title: '我的文档',
  tags: ['mdx', 'tutorial'],
};

// 导出默认布局组件
export default function Layout({ children }) {
  return <article className="prose">{children}</article>;
}
```

#### Fragment 语法

当需要返回多个元素但不想添加额外 DOM 节点时：

```jsx
<>
  第一段内容
  第二段内容
</>
```

### 3.3 MDX 编译工具

#### @mdx-js/mdx - 官方编译器

MDX 的核心编译器，由 Unified 团队维护。

```bash
npm install @mdx-js/mdx
```

```javascript
import { compile } from '@mdx-js/mdx';

const compiled = await compile('# Hello {name}', {
  // 编译选项
  jsx: false,      // true 则输出 JSX 而非函数调用
  pragma: 'React.createElement',
  pragmaFrag: 'React.Fragment',
});
```

**适用场景**：自定义构建流程、非框架项目、CLI 工具开发。

#### @mdx-js/react - React 集成

为 React 提供 MDX 组件上下文支持。

```bash
npm install @mdx-js/react
```

```jsx
import { MDXProvider } from '@mdx-js/react';
import Content from './content.mdx';

const components = {
  h1: (props) => <h1 className="title" {...props} />,
  code: (props) => <code className="highlighted" {...props} />,
};

function App() {
  return (
    <MDXProvider components={components}>
      <Content />
    </MDXProvider>
  );
}
```

#### @mdx-js/preact - Preact 集成

为 Preact 项目提供 MDX 支持，用法与 React 版本类似。

### 3.4 渲染引擎与框架集成

#### Astro - 零 JavaScript 渲染

Astro 是 MDX 的最佳搭档之一。默认情况下，Astro 会将 MDX 编译为静态 HTML，**不向客户端发送任何 JavaScript**。只有当你显式使用 `client:` 指令时，交互组件才会加载 JS。

```astro
---
// src/pages/docs/getting-started.mdx
import InteractiveChart from '../components/Chart.tsx';
---

# 快速开始

这是一段静态内容。

<!-- 只有这个组件会加载 JS -->
<InteractiveChart client:visible data={[1, 2, 3]} />
```

#### Next.js - App Router 支持

Next.js 13+ 的 App Router 原生支持 MDX：

```bash
npm install @next/mdx @mdx-js/loader
```

```javascript
// next.config.mjs
import createMDX from '@next/mdx';

const nextConfig = {
  pageExtensions: ['md', 'mdx', 'ts', 'tsx'],
};

export default createMDX()(nextConfig);
```

```jsx
// app/docs/[slug]/page.mdx
export const metadata = { title: '文档' };

# 文档内容

直接在 App Router 中使用 MDX 文件作为页面。
```

#### Gatsby - GraphQL 集成

```bash
npm install gatsby-plugin-mdx @mdx-js/react
```

```javascript
// gatsby-config.js
module.exports = {
  plugins: [
    {
      resolve: 'gatsby-plugin-mdx',
      options: {
        extensions: ['.mdx', '.md'],
      },
    },
  ],

```

#### Remix / React Router

通过 `@mdx-js/rollup` 插件集成：

```bash
npm install @mdx-js/rollup
```

```javascript
// vite.config.ts
import mdx from '@mdx-js/rollup';

```

### 3.5 编辑器支持

#### VS Code

- **MDX 扩展**（`silvenon.mdx`）：提供语法高亮、自动补全、错误检查
- **MDX Preview**：实时预览 MDX 文件渲染效果
- **TypeScript 支持**：通过 `tsconfig.json` 中的 `allowJs: true` 启用类型检查

推荐 VS Code 配置：

```json

  "editor.wordWrap": "on",

```

WebStorm 和 IntelliJ IDEA 通过内置的 MDX 插件提供支持，包括语法高亮和代码补全。

#### Vim / Neovim

通过 `nvim-treesitter` 的 MDX 语法解析器获得语法高亮和代码折叠支持。

### 3.6 remark / rehype 插件生态

MDX 基于 Unified 生态系统，可以使用 remark（Markdown AST）和 rehype（HTML AST）插件来扩展功能：

```javascript
// astro.config.mjs 示例
import remarkMath from 'remark-math';
import rehypeKatex from 'rehype-katex';
import remarkGfm from 'remark-gfm';
import rehypeSlug from 'rehype-slug';

export default defineConfig({
  markdown: {
    remarkPlugins: [remarkGfm, remarkMath],
    rehypePlugins: [rehypeSlug, rehypeKatex],
  },
});
```

**常用 remark 插件：**

| 插件 | 功能 |
|------|------|
| `remark-gfm` | GitHub 风格 Markdown（表格、任务列表、删除线等） |
| `remark-math` | 数学公式支持（LaTeX 语法） |
| `remark-directive` | 自定义指令语法（`:::warning` 等） |
| `remark-emoji` | emoji 短代码支持 |
| `remark-toc` | 自动生成目录 |

**常用 rehype 插件：**

| 插件 | 功能 |
|------|------|
| `rehype-katex` | 渲染数学公式 |
| `rehype-slug` | 为标题添加 `id` 属性 |
| `rehype-autolink-headings` | 为标题添加锚点链接 |
| `rehype-highlight` | 代码语法高亮 |
| `rehype-pretty-code` | 更美观的代码高亮（基于 Shiki） |

### 3.7 内容管理与 CMS

#### Contentlayer

将 MDX 文件转化为类型安全的数据层：

```bash
npm install contentlayer next-contentlayer
```

```javascript
// contentlayer.config.js
import { defineDocumentType, makeSource } from 'contentlayer/source-files';

const Post = defineDocumentType(() => ({
  name: 'Post',
  filePathPattern: '**/*.mdx',
  fields: {
    title: { type: 'string', required: true },
    date: { type: 'date', required: true },
  },
  computedFields: {
    slug: { type: 'string', resolve: (doc) => doc._raw.flattenedPath },
  },
}));

export default makeSource({ contentDirPath: './posts', documentTypes: [Post] });
```

#### Velite

Contentlayer 的现代替代品，支持 Zod schema 验证：

```bash
npm install velite
```

#### Payload CMS

支持 MDX 作为富文本字段类型的内容管理系统。

## 四、总结与建议

### 何时选择 MDX？

- 你的内容需要嵌入交互式组件
- 你在构建组件文档或技术文档
- 你的项目已经使用 React/Vue/Astro 等框架
- 你需要在 Markdown 中复用 UI 组件
- 你追求内容与代码的统一管理

### 何时继续使用 Markdown？

- 内容以纯文本为主，不需要交互
- 团队成员不熟悉 JSX 语法
- 项目对构建复杂度有严格限制
- 简单的 README 或文档

### 学习路径建议

1. **先掌握 Markdown 基础**：确保熟悉标准 Markdown 语法
2. **了解 JSX 基本概念**：不需要深入 React，理解 JSX 语法即可
3. **从简单开始**：先在 MDX 中使用现成的组件，再尝试自定义组件
4. **逐步扩展**：学习 remark/rehype 插件，扩展 MDX 功能
5. **实践出真知**：在实际项目中使用 MDX，在实践中积累经验

---

*MDX 不是要取代 Markdown，而是在 Markdown 的基础上提供了更多可能性。选择哪种格式，取决于你的内容需求。当文字不够表达时，MDX 就是你的答案。*

> 写作的本质是沟通。工具只是手段，重要的是你想传达什么。
